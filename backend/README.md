# CodeLens AI PR Review Assistant - Backend (Phase 4)

## Current Phase Capabilities
This phase adds lightweight diff parsing and rule-based risk analysis on top of GitHub PR fetching.

Implemented:
- `GET /` base info endpoint.
- `GET /health` health check endpoint.
- `POST /api/parse-pr-url` GitHub PR URL parser.
- `POST /api/review` fetches PR metadata and changed files from GitHub API.
- Diff parsing for patch hunks, added lines, and deleted lines.
- Rule-based risk analysis output: `risks`, `suggestions`, `risk_summary`.
- Optional `github_token` support for authenticated API requests.
- Patch truncation for each changed file (max 6000 chars).
- GitHub API error handling (404, 403, 429/rate limit, timeout/network).

Not implemented in this phase:
- AI summary generation and AI review suggestions.
- Database persistence.
- Automatic PR comments.
- Frontend integration.

## Supported Risk Types
- Hardcoded Secret
- Dangerous Function Usage
- Swallowed Exception
- Potential SQL Injection
- Dependency Change
- Configuration Change
- Auth/Permission Sensitive Change
- Large Change Set
- TODO/FIXME Marker
- Missing Tests (PR-level)

## Install Dependencies
```bash
pip install -r requirements.txt
```

## Run Backend
```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

## Endpoint Tests

### 1) Health Check
```bash
curl -X GET "http://127.0.0.1:8000/health"
```

### 2) Parse PR URL (Valid)
```bash
curl -X POST "http://127.0.0.1:8000/api/parse-pr-url" \
  -H "Content-Type: application/json" \
  -d "{\"pr_url\":\"https://github.com/owner/repo/pull/123\"}"
```

### 3) Parse PR URL (Invalid)
```bash
curl -X POST "http://127.0.0.1:8000/api/parse-pr-url" \
  -H "Content-Type: application/json" \
  -d "{\"pr_url\":\"https://example.com/not-a-pr\"}"
```

### 4) Rule-Based Review Without Token (Public Repo)
```bash
curl -X POST "http://127.0.0.1:8000/api/review" \
  -H "Content-Type: application/json" \
  -d "{\"pr_url\":\"https://github.com/owner/repo/pull/123\",\"github_token\":\"\",\"use_ai\":false}"
```

### 5) Rule-Based Review With Token
```bash
curl -X POST "http://127.0.0.1:8000/api/review" \
  -H "Content-Type: application/json" \
  -d "{\"pr_url\":\"https://github.com/owner/repo/pull/123\",\"github_token\":\"<YOUR_GITHUB_TOKEN>\",\"use_ai\":false}"
```

## Notes
- Public repositories can usually be accessed without token.
- For private repositories or frequent requests, pass `github_token` to reduce permission/rate-limit issues.
- Current review suggestions are generated by deterministic rules, not by any LLM.
- Risk `severity` indicates potential impact (`High` > `Medium` > `Low`).
- Risk `confidence` indicates rule match certainty (`High` means stronger pattern evidence).
- Never commit real `github_token` or API secrets into this repository.
