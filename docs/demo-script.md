# Demo 讲解脚本

本脚本面向七牛云 × XEngineer 暑期实训营作品演示，适合控制在 2 到 3 分钟内。建议录制时打开前端页面、后端运行终端，以及一个可稳定访问的公开 GitHub Pull Request 链接。

## 1. 开场：作品定位

大家好，我的作品是 `CodeLens AI PR Review Assistant`，这是一个面向开发者的 AI Pull Request 代码评审助手。

它不是一个简单的大模型问答页面，而是一个完整的 PR Review 工作流工具。用户输入 GitHub PR 链接后，系统会自动获取代码变更、做 Diff 解析、识别风险、生成 AI Review 建议，并输出可复制的 Markdown Review 报告。

## 2. 展示输入流程

首先我打开前端页面，可以看到这里支持输入 GitHub PR URL。

用户还可以按需填写 GitHub Token，用来减少 GitHub API 的 rate limit 影响。除此之外，还可以选择是否启用 AI Review。

这里我输入一个公开可访问的 GitHub PR 链接，然后点击“开始分析”。

## 3. 展示分析结果

分析完成后，页面会展示这次 PR 的基本信息，包括：

- PR 标题
- 作者
- 分支信息
- 变更文件数
- 新增和删除行数

接着系统会展示风险概览和风险详情。

这里的核心亮点是：系统不是只给一个笼统总结，而是会把风险项结构化展示出来，包括风险等级、命中位置、证据内容和修复建议。

## 4. 展示规则分析与 AI 结合

这个项目采用的是“规则分析 + AI 生成”的混合方案。

规则部分负责识别高确定性问题，比如硬编码密钥、危险函数、SQL 拼接、依赖变更、测试缺失等。

AI 部分负责基于 PR 上下文和规则结果，生成更自然的 PR 总结、风险分析和 Review 建议。

如果没有配置模型，或者模型调用失败，系统也会自动 fallback 到规则模式，保证主流程可用。

## 5. 展示 Markdown 报告输出

最后，这里可以看到系统还会生成一份 Markdown Review 报告。

这份报告可以直接复制，用在 GitHub PR 评论区、团队评审记录或者项目归档文档里。

这让工具不仅能“看”，还能真正服务真实开发场景。

## 6. 结尾总结

总结一下，这个作品的核心价值在于三点：

1. 它覆盖了从 PR 获取、Diff 解析到 Review 输出的完整链路。
2. 它通过规则分析和 AI 结合的方式，尽量降低误报和漏报。
3. 它最终输出结构化、可复制的 Markdown 报告，能够直接落地到真实开发协作流程中。

以上就是我的作品演示，谢谢观看。
