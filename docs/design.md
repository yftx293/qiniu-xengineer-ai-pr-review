# 系统设计文档

## 1. 作品定位

CodeLens AI PR Review Assistant 是一个 AI 辅助代码评审工具，面向需要快速理解 Pull Request 内容并发现潜在风险的开发者。

系统通过 GitHub API 获取 PR 基本信息和代码变更内容，再结合规则分析与 AI 生成能力，输出结构化 Review 结果。

## 2. 用户需求分析

目标用户主要包括：

- 需要快速 Review PR 的开发者
- 需要理解代码变更的新成员
- 希望提前发现风险代码的项目维护者
- 希望生成结构化 Review 建议的团队协作者

核心需求包括：

- 快速了解 PR 改了什么
- 找出可能有风险的代码
- 获得可操作的修改建议
- 生成可以复制到 GitHub PR 评论区的 Review 报告

## 3. 系统模块设计

系统计划分为五个模块：

### 3.1 前端交互模块

负责输入 GitHub PR 链接，展示 PR 信息、变更总结、风险列表和 Review 建议。

### 3.2 GitHub PR 获取模块

负责解析 PR 链接，并通过 GitHub REST API 获取 PR 基本信息、文件列表和 diff patch。

### 3.3 Diff 解析模块

负责提取变更文件、变更行数、patch 内容和关键上下文。

### 3.4 风险规则分析模块

负责识别高确定性的风险，例如硬编码密钥、危险函数、依赖变更、测试缺失、异常吞掉等。

### 3.5 AI Review 生成模块

负责基于 PR 上下文和规则分析结果生成自然语言总结、风险解释和 Review 建议。

## 4. 分析策略

系统采用“规则分析 + AI 生成”的混合方案。

规则分析用于识别明确风险，优点是稳定、快速、可解释。

AI 生成用于理解上下文并生成自然语言建议，优点是表达更自然，能够对复杂变更做总结。

为了降低误报，系统会将结果分为：

- High：明确风险，建议优先处理
- Medium：潜在风险，需要人工确认
- Low：代码质量建议，可作为优化参考

## 5. 未来扩展方向

后续可以扩展为：

- GitHub App，支持自动评论 PR
- 支持团队自定义 Review 规则
- 支持更多代码语言的专项检查
- 支持历史 Review 记录与团队知识库
- 支持 CI/CD 流程集成
